# Roadmap technique

## Sprint 0 - Setup

- installer Flutter SDK
- creer projet Flutter
- brancher Firebase (projets dev/prod)
- configurer CI simple (analyze + test)

## Sprint 1 - Fondations

- Auth Firebase (email/password)
- ecrans onboarding / connexion
- creation et liste de voyages
- couche repository + modeles de base

## Sprint 2 - Collaboration coeur

- membres / invitations
- module courses collaboratives
- module repas (planning)

## Sprint 3 - Mobilite + finance

- module covoiturage
- module depenses + soldes
- premiers indicateurs d'activite

## Sprint 4 - Stabilisation

- regles Firestore robustes
- notifications push (rappels repas/courses)
- QA fonctionnelle + tests widget

## Definition of Done (MVP)

- un voyage peut etre cree et partage avec >=3 participants
- les participants peuvent gerer courses + repas + depenses
- le solde final par personne est calcule correctement
- les regles Firestore bloquent les acces non membres
